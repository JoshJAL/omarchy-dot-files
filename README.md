# Omarchy Dotfiles

Version-controlled configuration for my [Omarchy](https://omarchy.org/) (Arch + Hyprland) setup.
The goal: never lose custom settings — keybindings, idle/lock timings, themes, bar layout —
across `omarchy update` or a full reinstall.

## How it works

This is a **bare git repository** with its work-tree set to `$HOME`. Config files are tracked
**in place** under `~/.config/` — nothing is moved and there are no symlinks. Omarchy keeps
reading the real files exactly as before; git just version-controls them.

> Why no symlinks? `omarchy refresh <app>` *replaces* config files. A symlink-based dotfiles
> setup would break the moment you ran a refresh. Tracking in place avoids that entirely.

The repo lives at `~/.dotfiles` and is driven through a shell alias:

```bash
alias dotfiles='git --git-dir=$HOME/.dotfiles --work-tree=$HOME'
```

(Added to `~/.bashrc`.)

## Everyday usage

```bash
dotfiles status                       # see what changed
dotfiles add ~/.config/hypr/bindings.conf
dotfiles commit -m "tweak keybinding"
dotfiles push                         # back up to GitHub
```

Restore a single file after a bad update:

```bash
dotfiles checkout -- .config/hypr/hypridle.conf
```

## What's tracked

| Area | Path |
|------|------|
| Hyprland (keybindings, idle/lock, monitors, look & feel) | `~/.config/hypr/` |
| Status bar | `~/.config/waybar/` |
| App launcher | `~/.config/walker/` |
| Notifications | `~/.config/mako/` |
| On-screen display | `~/.config/swayosd/` |
| Terminals | `~/.config/alacritty/`, `~/.config/kitty/`, `~/.config/ghostty/` |
| Omarchy hooks / branding / extensions | `~/.config/omarchy/` |

## What's intentionally NOT tracked

Excludes live in `~/.dotfiles/info/exclude`:

- **Wallpaper / image / video binaries** — kept on disk, just not in git (hundreds of MB).
- **`~/.config/omarchy/themes/`** — each installed theme is its own upstream git repo, so they
  are not duplicated here. Instead their source URLs are recorded in
  [`~/.config/omarchy/themes-installed.txt`](.config/omarchy/themes-installed.txt) for one-shot
  reinstall (see below).
- **`~/.config/omarchy/current/`** — live theme-state symlinks, regenerated by `omarchy theme set`.
- **`~/.config/omarchy/branding/about.txt`** — rewritten on every shell launch.
- **`*.bak.*`** — timestamped local backups.
- **`~/.bashrc`** — kept out of git because it contains plaintext secrets.

`status.showUntrackedFiles=no` is set so `dotfiles status` reports only tracked files instead of
the entire home directory.

## Restoring on a fresh machine

After installing Omarchy on a new machine:

```bash
# 1. Clone this repo as a bare repo
git clone --bare git@github.com:JoshJAL/omarchy-dot-files.git $HOME/.dotfiles

# 2. Define the alias (also add it to ~/.bashrc)
alias dotfiles='git --git-dir=$HOME/.dotfiles --work-tree=$HOME'

# 3. Don't show all of $HOME as untracked
dotfiles config status.showUntrackedFiles no

# 4. Check out the tracked files into $HOME
#    (back up any conflicting stock files first if checkout complains)
dotfiles checkout main

# 5. Reinstall all themes from the manifest
grep -v '^#' ~/.config/omarchy/themes-installed.txt | awk '{print $2}' | xargs -rn1 omarchy theme install
```

## Why this is safe alongside `omarchy update`

Omarchy sources its own defaults from `~/.local/share/omarchy/` **first**, then sources user
files from `~/.config/` **after** — so user settings always win and are never overwritten by an
update. Migrations only make surgical edits, and `omarchy update` snapshots the system before
running. This repo adds full version history on top of all that.
